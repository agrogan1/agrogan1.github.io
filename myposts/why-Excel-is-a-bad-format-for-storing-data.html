Why Excel Is A Bad Format For Storing Data

Andy Grogan-Kaylor

14 Jan 2021 15:58:21

Setup

. clear all

Simulate Data

. set obs 100 // 100 observations
number of observations (_N) was 0, now 100

. generate Q1 = runiformint(1,4) // randomly simulated Question 1

. label variable Q1 "How often do you feel happy?" // add variable label

. label define Likert1 1 "Never" 2 "Rarely" 3 "Sometimes" 4 "Often" // define value label

. label values Q1 Likert1 // add value label to Q1

Descriptive Statistics and Bar Graph

Notice how the descriptive statistics and graph are informative in that they contain information on the variable name and value label. These help us to get an intuitive sense of the information in the data.

. tabulate Q1

  How often │
do you feel │
     happy? │      Freq.     Percent        Cum.
────────────┼───────────────────────────────────
      Never │         20       20.00       20.00
     Rarely │         24       24.00       44.00
  Sometimes │         29       29.00       73.00
      Often │         27       27.00      100.00
────────────┼───────────────────────────────────
      Total │        100      100.00

. graph bar, over(Q1) scheme(michigan)

. graph export mybar1.png, width(500) replace
(file mybar1.png written in PNG format)

Bar Graph

Now Simulate Removing The Information That Is Lost When Saving to Excel

. label variable Q1 // drop variable label

. label values Q1 // drop value label

Descriptive Statistics and Bar Graph

Notice how the descriptive statistics and graph are much less informative. The information on variable label and value label will have to be added back into the data when preparing a report for dissemination.
